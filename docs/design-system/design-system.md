# EcoLens AI Design System

## 1. Product Vision
EcoLens AI is a premium, dark-mode-first SaaS experience for carbon footprint analysis from receipts. The design system should feel modern, calm, trustworthy, and enterprise-ready while supporting highly polished product interactions.

## 2. Design Principles
- Clarity over decoration
- Premium and minimal, not noisy
- Confidence through consistency
- Motion should feel smooth and purposeful
- Dark mode is the default experience
- Accessibility is a core requirement

## 3. Visual Direction
- Inspired by Linear, Stripe, Vercel, Apple, Notion, and GitHub
- Glassmorphism with soft depth
- Rounded surfaces and subtle blur
- Strong hierarchy and restrained color usage
- Responsive behavior across desktop, tablet, and mobile

## 4. Design System Goals
- Create a reusable foundation for all future screens
- Standardize spacing, typography, color, motion, and components
- Ensure scalability for product growth and multiple teams
- Align visual language with a premium AI SaaS brand

## 5. Core Foundations
### Color
- Primary: Emerald (#10B981)
- Secondary: Ocean Blue (#3B82F6)
- Accent: Purple (#8B5CF6)
- Success: Green
- Warning: Orange
- Danger: Red
- Neutral: Slate Gray

### Typography
- Font family: Inter
- Hierarchy includes Display, H1, H2, H3, Subtitle, Body, Caption, and Button

### Spacing
- Use a consistent spacing scale based on 4px increments

### Radius
- Use rounded surfaces with a layered radius system

### Shadows
- Soft elevation for depth without harsh contrast

### Motion
- Use subtle, fast transitions with clear easing curves

## 6. Design Tokens Structure
- Color tokens
- Typography tokens
- Spacing tokens
- Radius tokens
- Shadow tokens
- Motion tokens
- Layout tokens

## 7. Component Categories
- Actions: buttons, links, toggles
- Inputs: search, upload, dropdown, date picker, checkbox, radio, switch, slider
- Data display: cards, tables, charts, badges, lists
- Navigation: navbar, sidebar, breadcrumb, profile menu, notification panel
- Feedback: alerts, loaders, empty states, toasts

## 8. Layout System
- Desktop-first, responsive by breakpoint
- Flexible dashboard grid with consistent gutters
- Content density should remain premium and readable

## 9. Accessibility Guidelines
- Minimum contrast targets for text and controls
- Keyboard-friendly interaction patterns
- Focus states should be visible and consistent
- Clear labels and accessible names for all form controls

## 10. Implementation Notes for Future Frontend Work
- Tokens should be centralized and reusable
- Components should follow atomic design principles where practical
- Consistent spacing and typography rules should be enforced across all screens
